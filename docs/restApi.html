REST API

Para el API, las opciones si bien pueden ser muchas ya se tenía previsto usar javascript como lenguaje principal para el desarrollo.

En terminos de levantar el servidor para los request, se se decide que también se iba a utilizar Express JS. Ambas decisiones vienen dadas por la experiencia previa con estas herramientas por lo que sería una ventaja empezar con estas. Cabe destacar la facilidad de la herramienta para integrar servicios que serán necesarios para el desarrollo del sistema.

Para el proyecto se utiliza Express JS por su facilidad de habilitar diferentes tipos de request, lanzar mensajes de error y habilitar CORS. Posteriormente se decide migrar el proyecto a TypeScript que es una plataforma que brinda más beneficios en terminos de herramientas y estándares de código. Además tiene muchas ventajas para usar diferentes tipos de paquetes de NPM y sus respectivos tipos, y la documentación sobre este es bastante extensa.
