Estándares de Programación

Para la sección de estándares, el equipo decidió principalmetne seguir los siguientes:

* Documentación de código: Dentro de los estandares el primero y de los más importantes es la correcta documentación, tanto como la documentación interna del código así como el uso de herrramientas como Swagger.

* Manejo de archivos: Todos los archivos se usaran con el nombre en inglés y que sea explicito con lo que se contenga en este. Los archivos deben de empezar con letra minúscula.

* Uso de variables para usuario: Las variables de usaruio como las de las bases de datos están en español, esta decisión es principalmente porque al desarrollarlo en un país de habla en español, lo hacemos más accesible para la comunidad local.

* Uso de Github: Los commits que se harán en el GitHub deben de ser en inglés y explicitos con lo que se haya modificado, además se espera que al menos si no es muy explicito el mismo se empiece cada commit con las tags [ADD], [UPDATE] , [DELETE].

* Uso de Notion: Notion se usará como centro de informaciónd de avance del proyectop, centro donde están links de importancia, tareas y avance de las mismas.
