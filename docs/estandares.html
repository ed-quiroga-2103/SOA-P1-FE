Estándares de Programación

Para la sección de estándares en el desarrollo, el equipo decidie plantear lo siguiente:

* Documentación de código: Dentro de los estandares, el primero y de los más importantes es la correcta documentación, tanto a nivel interno, como con el uso de herrramientas externas como Swagger.

* Manejo de archivos: Todos los archivos se usaran con el nombre en inglés, el cual describa el contenido del mismo. Los archivos deben de empezar con letra minúscula, excepto los componentes de ReactJS, los cuales inician con mayúscula.

* Manejo de Datos: Los datos que se almacenen ya sea de usuarios o canciones, deberán tener sus nombres en inglés para mantener la congruencia con el primer punto especificado en esta sección.

* Uso de Github: Los commits que se harán en el GitHub deben de ser en inglés y explicitos con lo que se haya modificado, además se espera que el mensaje del commit inicie con un verbo descriptivo de qué se realizó en la modificación.
