Motor de Bases de Datos

Para el motor de la base de datos se plantearon dos opciones principalmente: MySQL y MongoDB, las cuales se determinaron por la familiaridad con los sistemas, y porque presentan dos modelos de almacenamiento diferentes; MySQL en tablas, y MongoDB en documentos.

En este caso se decide optar por MongoDB por las siguientes razones:

* Previamente se había escogido TypeScript y Node.js que tienen facilidad para trabajar con MongoDB.

* MongoDB facilita la creación de bases de datos ya que no necesitan script y, como los documentos se guardan en formato JSON resulta más maleable y fácil de modificar que una base de datos SQL.

* Como se verá en el siguiente punto, MongoDB además posee un fácil storage en cloud por medio del servicio MongoDB Atlas.

* La forma de hacer requests a la base de MongoDB por medio de Express JS también resulta en trabajo más sencillo haciendo que el desarrollo se facilite para los realizar los request.
