Motor de Bases de Datos

Para el motor de la base de datos se tuvieron dos opciones principalmente: MySQL y MongoDB. Estas dos opciones son importantes porque presentan dos mundos diferentes en las bases de datos de forma que tenemos una base SQL y otra No-SQL.

En este caso se decide optar por MongoDB por las siguientes razones:

* Previamente se había escogido typescript y Node.js que tienen plugins para Mongo

* Mongo DB tiene una fácil forma para crear bases de datos que no necesitan script y, como se guarda en formato JSON es más maleable y fácil de modificar que una base de datos en SQL.

* Como se verá en el siguiente punto Mongo además posee un fácil storage en cloud por medio de Mongo Atlas.

* La forma de hacer request a la base de Mongo por medio de typescript también hace más sencillo el trabajo haciendo que el desarrollo se facilite más para los request.
